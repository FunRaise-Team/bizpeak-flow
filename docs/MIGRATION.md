# BizPeak Flow — 轉移與嵌入規劃 v0.1

> 原則：**不搬家、先接管** — 導入失敗的第一名原因是要求大家換工具。信箱照用、Teams 照用、報價單系統照用；系統把追蹤層疊在既有工具之上、只退役「補洞用」的表格與登記簿。
> 互動版（雙態滑動圖）見提案頁第 7 節；現用軟體清單以本週訪談最終確認。

## 1. 工具去留判定

| 工具 | 現況角色 | 去向 | 資料怎麼搬 |
|------|----------|------|-----------|
| Outlook 信箱 | 回簽與往來信、靠人看到 | **照用** — 回簽識別代理自動讀 | 回簽檔案自動落合約庫、不用搬 |
| Teams | 口頭轉達與訊息 | **照用** — 通知與簽核卡片、可直接操作 | 操作記錄落事件庫 |
| 報價單系統 | 產出後斷鏈 | **照用** — 輸出直接建 C0 | 報價資料自動帶入合約庫 |
| NuEIP | 內部簽核 | **降為可選接口** — 簽核引擎自建 | 政策要求的流程走接口回寫 |
| 網銀・會計系統 | 財務逐筆對明細 | **照用** — 入帳明細餵對帳代理 | 實收結果回寫款項庫 |
| Excel 逾期表 | 月底結帳才更新 | **退役** — 逾期看板取代 | 歷史資料一次性導入款項庫 |
| Excel 現金流表 | 手動彙整預計付款 | **退役** — 款項庫彙總自動產出 | 歷史資料一次性導入 |
| 用印登記簿 | 紙本登記、不可查 | **退役** — 用印佇列取代 | 掃描歸檔事件庫 |
| 同仁自製小工具 | 各自為政、格式不一 | **卡帶化** — 四件套接入（L0 起步） | 資料契約登記卡帶註冊庫 |

## 2. 存量資料三步導入（雙軌不斷線）

1. **存量導入**：AI 讀歷史合約與報價（檔案、信件）、抽取欄位建檔、人逐筆確認 — 半自動、無人工謄打
2. **雙軌兩週**：新案 100% 進系統；進行中舊案跑完或搬入；既有表格凍結為唯讀、當對照組
3. **退役與卡帶化**：訪談確認無遺漏後舊表退場；個人小工具走 L0 卡帶註冊、好用的升 L1

## 3. 嵌入現有工作流（各角色一天的變化）

| 角色 | 之前 | 之後 |
|------|------|------|
| 業務 | 記得追回簽、記得催款、Excel 記進度 | 信箱照收、Teams 收到「已起草好的催收信」按一下送出；卡關自動浮現 |
| 行政 | 紙本登記用印、被問進度 | 用印佇列自動排今天的件、完成即通知業務 |
| 財務 | 看信轉知、月底逐筆對帳半天 | 回簽 5 分鐘內收到通知；對帳代理給「建議打勾」清單、一鍵確認 |
| 管理層 | 月底才知道現金流全貌 | 現金流量預測即時、逾期 D+1 浮現、週報自動送 |
